# Kubernetes Cluster Configuration

This directory contains the Terraform code to configure the Kubernetes cluster after it has been provisioned. It handles namespaces, core services (ArgoCD, Cilium LoadBalancer), and GitOps bootstrapping.

## Overview

The application layer performs the following tasks:
- Creates the `argocd` namespace.
- Installs **ArgoCD** via the official Helm chart (`v9.5.9`).
- Configures **Cilium L2 Announcement** and **LoadBalancer IP Pools** to enable external access to services.
- (Optional) Bootstraps the cluster using the **ArgoCD App-of-Apps** pattern.
- (Optional) Provisions credentials for private Git repositories.

## Prerequisites

- A running Talos/Kubernetes cluster (provisioned via `terraform/proxmox`).
- The `kubeconfig` file generated by the infrastructure layer.

## Configuration

Configuration is managed via the `terraform.tfvars` file. Key variables include:

| Variable | Description | Example |
| :--- | :--- | :--- |
| `kubernetes_config_path` | Path to your kubeconfig | `../proxmox/kubeconfig` |
| `Kubernetes_config_context` | Context name in kubeconfig | `admin@talos` |
| `cilium_load_balancer_ip_range_start` | Start of the LB IP pool | `10.0.0.20` |
| `cilium_load_balancer_ip_range_stop` | End of the LB IP pool | `10.0.0.50` |
| `install_argocd_app_of_apps` | Enable GitOps bootstrap | `false` (default) |

## GitOps - App of Apps

This project is designed to follow the GitOps "App of Apps" pattern. 
- When `install_argocd_app_of_apps` is set to `true`, Terraform will create an ArgoCD Application that tracks the `apps/` directory in your repository.
- You should place your application manifests in `apps/base/` or `apps/production/` to have them automatically managed by ArgoCD.

## Usage

1.  Copy the example variables:
    ```bash
    cp terraform.tfvars.example terraform.tfvars
    ```
2.  Initialize and apply:
    ```bash
    terraform init
    terraform apply
    ```

## Verification

Check that services are getting External IPs:
```bash
kubectl get svc -A
```

## Accessing ArgoCD

After a successful apply, you can retrieve the initial admin credentials using Terraform:

1. **Get the URL & Username:**
   ```bash
   terraform output argocd_url
   terraform output argocd_username
   ```
2. **Get the Password:**
   ```bash
   terraform output -raw argocd_initial_admin_password
   ```

3. **Login:**
   Open your browser and navigate to the URL. Use `admin` and the password retrieved above to log in.

> [!TIP]
> Since we use a local IP (`10.0.0.20`), you might need to add `10.0.0.20 argocd.local` to your local `/etc/hosts` file to use the domain name.
