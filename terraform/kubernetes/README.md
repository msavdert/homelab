# Kubernetes Cluster Configuration

This directory contains the OpenTofu (Terraform) code for managing the internal state of the Kubernetes cluster once it has been bootstrapped.

## Overview

The Kubernetes layer is responsible for:
- Creating namespaces for system and application components.
- Managing sensitive information via Kubernetes Secrets.
- Deploying core services using Helm.
- Initializing the GitOps workflow by installing ArgoCD and the "App of Apps" bootstrap chart.

## File Details

- **`providers.tf`**: Configures the `kubernetes` and `helm` providers. It typically consumes the `kubeconfig` generated by the infrastructure layer.
- **`namespaces.tf`**: Defines `kubernetes_namespace_v1` resources. By default, it ensures the `argocd` namespace exists.
- **`secrets.tf`**: Manages secrets such as Docker registry credentials or Git repository SSH keys needed by ArgoCD.
- **`helm_releases.tf`**: 
    - **ArgoCD**: Installs the ArgoCD operator to manage application lifecycles.
    - **Cilium LB Config**: A local Helm chart that configures Cilium's L2 Announcement and IP Address Pools (LoadBalancer functionality).
    - **App of Apps**: A bootstrap pattern that tells ArgoCD to watch a specific directory in this repository for application definitions.
- **`helm_charts/`**:
    - `cilium-lb-config`: Local chart for Cilium-specific CRDs.
    - `app-of-apps`: Local chart that defines `Application` resources for ArgoCD.

## Integration with Infrastructure

This layer depends on the `terraform/proxmox` layer having completed successfully. It requires a valid `kubeconfig` to communicate with the cluster. 

## Configuration

Key variables in `variables.tf` include:
- `argocd_helm_values`: Custom values for the ArgoCD Helm chart.
- `cilium_load_balancer_ip_range_start/stop`: The IP range for Kubernetes LoadBalancer services.
- `argocd_app_of_apps_source`: The Git repository URL and path for the App-of-Apps pattern.

## Usage

1.  Ensure your `KUBECONFIG` environment variable is set or the `kubeconfig` file is present as expected by `providers.tf`.
2.  Initialize the workspace:
    ```bash
    tofu init
    ```
3.  Apply the configuration:
    ```bash
    tofu apply
    ```

## GitOps Workflow

Once `app-of-apps` is deployed, ArgoCD will take over. Any changes made to the `apps/` directory in the Git repository will be automatically synchronized to the cluster.
